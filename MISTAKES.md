Fixed by restoring fetch-based API calls and route handlers.

- 2026-04-15: I initially created the PGlite directory using `dirname(dataDir)`, which produced transient DB roots and broke table visibility between operations. Fixed by creating `dataDir` directly and adding a repository reset helper for tests.
- 2026-04-17: `AsyncResult.isWaiting()`, `isSuccess()`, `isFailure()` from Effect 4.x access `result._tag` directly and throw "Cannot read properties of null (reading '_tag')" when passed null. Root cause: `useAtomValue` from `@effect/atom-solid` initializes its internal signal with `null` before the atom mounts to the registry. **FIXED**: Added `useAtomInitialValues([[Atoms.dashboardSnapshotAtom, AsyncResult.initial(true)]])` in `DashboardProvider` to pre-seed the atom with a valid `AsyncResult.Initial` state before first render. This ensures `useAtomValue` never returns null, eliminating the crash at the source without wrapper functions.
